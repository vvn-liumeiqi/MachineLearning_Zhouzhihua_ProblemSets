# 问题描述（分析训练集和测试集的不同分法对模型的影响）
1. 核心任务是：选择两个UCI数据集，分别用10折交叉验证法和留一法评估逻辑回归模型的错误率，并对结果进行比较分析。
2. 将数据集随机分成10个大小相近的子集（“折”），依次用其中9个作为训练集、1个作为测试集，重复10次，每次使用不同的测试集，最终取10次测试结果的平均值
   原始数据 → 随机划分为10份
   第1次：用第2-10份训练，第1份测试
   第2次：用第1、3-10份训练，第2份测试
   ...
   第10次：用第1-9份训练，第10份测试
   最终错误率 = 10次错误率的平均值
3. 留一法是k折交叉验证的一种特例——k等于样本总数N。每次用N-1个样本训练，剩下的1个样本测试，重复N次，每个样本恰好作为测试集一次

## 数据集Transfusion
1. 经典的机器学习数据集——Blood Transfusion Service Center Data Set（输血服务中心数据集），它包含748名献血者的数据。
2. 如何通过分析历史行为来预测未来的献血意愿
3. 数据一共包含5列，每列的含义如下：
R	最近一次献血相隔的月份数	数值越小，表示近期刚献过血
F	总献血次数	累计次数
M	总献血量 (单位：毫升 c.c.)	累计献血量。需要注意，它的数值和方差会非常大
T	第一次献血距今的月份数	表示成为献血者的时长
y (目标)	2007年3月是否献血	1 表示“是”，0 表示“否”。这是我们要预测的目标列
数据的特点与用途
这个数据集在教学中很常用，有几个显著特点：
数据不平衡：大约 76% 的样本属于“未献血”类别 (0)，只有 24% 是“已献血”类别 (1)。这一点在进行分类时需要注意。
数值范围差异大：M 列（总献血量）的数值远大于其他列，例如样本中经常出现 12500 这样的数值，而 R 列的单位只是“几个月”。
